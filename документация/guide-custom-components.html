Кастомные компоненты [guide]

Утилитарные классы Photon позволяют создавать уникальные компоненты без написания собственного CSS. Комбинируйте pt-* классы для получения нужного результата.

Принцип композиции

Каждый pt-класс отвечает за одно CSS-свойство. Комбинируя их на одном элементе, вы строите сложный визуальный компонент без единой строчки кастомного CSS.

<!-- Пример композиции -->
<div class="pt-flex pt-items-center pt-justify-center pt-w-32 pt-h-32 pt-bg-primary pt-rounded-lg pt-shadow-lg">
  Компонент из утилит
</div>

Создание кнопок

Базовые стили кнопок можно расширять с помощью pt-утилит.

Кнопка с иконкой

<button class="btn btn--primary pt-flex pt-items-center pt-gap-2">
  <svg width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
    <path d="M5 12h14M12 5l7 7-7 7"/>
  </svg>
  <span>Отправить</span>
</button>

Группа кнопок

<div class="pt-flex">
  <button class="btn btn--primary" style="border-radius:8px 0 0 8px;">Левый</button>
  <button class="btn btn--primary" style="border-radius:0;border-left:1px solid rgba(0,0,0,0.2);">Центр</button>
  <button class="btn btn--primary" style="border-radius:0 8px 8px 0;border-left:1px solid rgba(0,0,0,0.2);">Правый</button>
</div>

Outline-кнопка с кастомным цветом

<button class="btn pt-border-2 pt-border-accent pt-text-accent pt-bg-transparent pt-rounded-lg pt-px-4 pt-py-2 pt-font-medium pt-transition">
  Кастомная
</button>

Создание модалок

Заголовок

Содержимое модального окна. Нажмите на фон или крестик, чтобы закрыть.

<div style="display:none;position:fixed;inset:0;z-index:9999;background:rgba(0,0,0,0.6);align-items:center;justify-content:center;" onclick="if(event.target===this)this.style.display='none'">
  <div class="pt-bg-surface pt-rounded-xl pt-shadow-xl pt-p-6 pt-max-w-md pt-w-full" onclick="event.stopPropagation()">
    <div class="pt-flex pt-items-center pt-justify-between pt-mb-4">
      <h2 class="pt-text-xl pt-font-bold">Заголовок</h2>
      <button class="pt-bg-transparent pt-border-none pt-text-2xl pt-cursor-pointer" style="color:var(--pk-muted);" onclick="this.closest('div[style]').parentElement.style.display='none'">&times;</button>
    </div>
    <p class="pt-text-secondary pt-mb-6">Содержимое.</p>
    <div class="pt-flex pt-justify-end pt-gap-3">
      <button class="btn" onclick="this.closest('div[style]').parentElement.style.display='none'">Отмена</button>
      <button class="btn btn--primary">Подтвердить</button>
    </div>
  </div>
</div>

Best practices

DRY — Don't Repeat Yourself

Если вы используете одну и ту же комбинацию pt-классов в нескольких местах, создайте кастомный CSS-класс:

/* Вместо повторения утилит */
.my-card {
  background: var(--pk-surface);
  border-radius: 12px;
  padding: 24px;
  box-shadow: 0 4px 20px rgba(0,0,0,0.1);
}

/* Использование в HTML */
<div class="my-card">...</div>

Именование классов

Используйте осмысленные имена на английском (или на русском, если проект внутренний). Придерживайтесь БЭМ или аналогичной методологии для сложных компонентов.

/* Хорошо */
.profile-card { }
.profile-card__avatar { }
.profile-card__name { }
.profile-card--highlighted { }

/* Плохо */
.block1 { }
.my-custom { }
.card-style-01 { }

Извлечение в CSS

Когда прототип готов, извлеките повторяющиеся комбинации утилит в отдельные CSS-классы. Это улучшит читаемость HTML и упростит поддержку.

/* Этап 1: HTML с утилитами (прототип) */
<button class="pt-bg-primary pt-text-white pt-px-6 pt-py-2 pt-rounded-lg pt-font-semibold pt-transition pt-hover-bg-primary-dark">
  Кнопка
</button>

/* Этап 2: Извлечение в CSS */
.btn-custom {
  background: var(--pk-primary);
  color: #fff;
  padding: 10px 24px;
  border-radius: 8px;
  font-weight: 600;
  transition: 0.2s ease;
}
.btn-custom:hover {
  background: var(--pk-primary-dark);
}

/* Этап 3: HTML с кастомным классом */
<button class="btn-custom">Кнопка</button>
