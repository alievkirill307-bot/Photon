Тема и кастомизация [pt]

Photon-tw предоставляет встроенную поддержку тёмной темы, адаптивных префиксов и комбинированных модификаторов. Для кастомизации цветовой палитры достаточно переопределить CSS-переменные --pk-*. Всё работает «из коробки» — просто подключите photon-kit.js.

Тёмная тема

Модификатор pt-dark:* применяет стили только тогда, когда страница (или родительский элемент) имеет класс .theme-dark или .dark. Это позволяет создавать компоненты, адаптирующиеся к обеим темам без дублирования кода.

Светлая тема

pt-bg-white pt-text-black

Тёмная тема

pt-dark:bg-gray-900 pt-dark:text-white

<div class="pt-flex pt-gap-4">
  <div class="pt-bg-white pt-text-black pt-p-4 pt-rounded-lg pt-shadow">
    <p class="pt-font-bold pt-mb-1">Светлая тема</p>
  </div>
  <div class="pt-bg-gray-900 pt-text-white pt-p-4 pt-rounded-lg pt-shadow-lg">
    <p class="pt-font-bold pt-mb-1">Тёмная тема</p>
  </div>
</div>

Адаптивные префиксы

Префиксы pt-sm:*, pt-md:*, pt-lg:* применяют класс только при достижении минимальной ширины экрана. Это позволяет легко создавать адаптивные макеты.

Префикс CSS-медиазапрос Мин. ширина Целевые устройства
pt-sm:*@media (min-width: 576px)576pxПланшеты, крупные телефоны
pt-md:*@media (min-width: 768px)768pxНебольшие десктопы, планшеты landscape
pt-lg:*@media (min-width: 992px)992pxШирокие десктопы

Пример: блок меняет цвет фона на каждом брейкпоинте.

Изменяет цвет на разных экранах

pt-bg-primary → pt-sm:bg-success → pt-md:bg-warning → pt-lg:bg-error

<div class="pt-bg-primary pt-sm:bg-success pt-md:bg-warning pt-lg:bg-error pt-text-white pt-p-4 pt-rounded pt-transition pt-text-center pt-font-bold">
  Изменяет цвет на разных экранах
</div>
  • pt-bg-primary — до 575px (по умолчанию)
  • pt-sm:bg-success — от 576px, зелёный
  • pt-md:bg-warning — от 768px, жёлтый
  • pt-lg:bg-error — от 992px, красный

Комбинирование модификаторов

Модификаторы можно выстраивать в цепочку: pt-dark:pt-hover:bg-primary-600. Движок парсит классы слева направо и генерирует правильный CSS-селектор с несколькими условиями.

Пример кнопки с комбинацией pt-dark:pt-hover:bg-primary-400:

<button class="pt-bg-primary pt-text-white pt-px-4 pt-py-2 pt-rounded pt-hover:bg-primary-600 pt-dark:pt-hover:bg-primary-400 pt-transition pt-cursor-pointer">
  Наведи меня (в тёмной теме)
</button>

Правила комбинирования:

  • pt-hover:* — при наведении курсора (:hover)
  • pt-focus:* — при фокусе (:focus)
  • pt-active:* — при активном нажатии (:active)
  • pt-dark:* — в тёмной теме (класс .theme-dark на родителе)
  • pt-sm:* / pt-md:* / pt-lg:* — на разных разрешениях
  • Любые два можно объединить: pt-dark:pt-hover:bg-primary-600, pt-sm:pt-hover:pt-text-lg

Отключение движка

Если на странице не нужна утилитарная генерация Photon-tw, просто не подключайте photon-kit.js или не вызывайте initPT(). Движок не будет запущен, и pt-* классы останутся без эффекта (как обычные неизвестные классы).

<!-- Движок НЕ будет запущен -->
<!-- <script src="photon-kit.js"></script> -->

Если нужно отключить только MutationObserver, но сохранить однократную генерацию при загрузке:

<script>
// Однократная генерация без наблюдения за изменениями
if (typeof initPT === 'function') {
  initPT({ observe: false });
}
</script>

Производительность

Photon-tw использует MutationObserver для отслеживания появления новых pt-* классов в DOM. По умолчанию наблюдение ведётся за всем документом, но можно ограничить область наблюдения конкретным контейнером, чтобы снизить нагрузку на динамических страницах.

Рекомендации для оптимальной производительности:

  • Ограничьте root-элемент — передайте контейнер в опциях initPT({ root: document.getElementById('app') })
  • Группируйте DOM-изменения — движок использует debounce, но частая мутация большого поддерева может вызвать задержки
  • Статичные страницы — используйте initPT({ observe: false }) для однократного прохода без подписки на мутации
  • Избегайте циклов — не добавляйте pt-* классы внутри обработчиков, которые срабатывают от изменений стилей
// Наблюдать только за #app
initPT({ root: document.getElementById('app') });

// Без MutationObserver — однократный проход
initPT({ observe: false });

// Полная конфигурация
initPT({
  root: document.body,
  observe: true,
  prefix: 'pt-'
});
